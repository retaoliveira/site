Projetos

Etapas essenciais para desenvolvimento de um projeto de suporte à decisão que contemple análise de dados

  1. Definição do problema - O primeiro e mais crítico passo é delinear as questões que pretende abordar por meio da análise de dados e desenhar hipóteses a partir da caracterização do problema.
  2. Coleta de dados - A coleta de dados é um passo muito crucial e não tão fácil como parece. O processo requer tempo e esforço. Nenhum conjunto de dados contém dados como se espera e envolve pesquisa, arranjos, reordenações e montagem final.
  3. Limpeza de dados - Se quiser que os seus resultados sejam consistentes, deve certificar-se de que a limpeza dos dados foi feita corretamente. Na essência, a limpeza de dados elimina dados desnecessários e duplicados da coleta de dados.
  4. Análise dos dados - Nesta fase, tem de detectar tendências e padrões nos dados coletados, agrupá-los em conformidade, e compreender o comportamento dos dados para que seja possível caracterizar o processo em investigação.
  5. Modelar os dados - Nesta fase, os dados são divididos em duas partes - uma para formação e desenvolvimento de modelos, e a outra para testes.
  6. Otimização e implementação do modelo - Nesta etapa, o modelo é implementado visando acurácia e eficiência das análises.

Ideias e Tópicos do Projeto Top R

Por agora, é bastante evidente que a linguagem de programação R tem um imenso potencial para aumentar os seus conhecimentos em Ciência e Análise de Dados. Na secção seguinte, discutiremos alguns dos tópicos de projetos de R mais tendenciais que pode utilizar para dominar as suas competências em Machine Learning and Data Science.

1. Análise de Sentimento

A análise dos sentimentos é o processo de análise das palavras para determinar opiniões e sentimentos que têm diferentes polaridades - positivas, negativas ou neutras. O método também se baseia nos nomes polarity detection e opinion mining. Neste tipo de classificação, os dados (sentimentos) são categorizados em diferentes classes; estas classes podem ser binárias (positivas e negativas), neutras ou múltiplas (felizes, tristes, zangadas, e assim por diante).

Então, qual é a sua utilidade? Bem, o processo de análise dos sentimentos pode ser utilizado para determinar a natureza das opiniões reflectidas nos websites, feeds das redes sociais, documentos, etc. O projeto de análise dos sentimentos pode ser construído em “R”, utilizando os conjuntos de dados do pacote “janeaustenr”.

2. Análise de dados de Uber

Uma componente crucial da Aprendizagem Automática é a narrativa de dados; ajuda as empresas a compreenderem os antecedentes e o contexto de várias operações. A visualização de dados ajuda as empresas a compreender conjuntos de dados complexos, o que, por sua vez, ajuda-as a tomar decisões.

O Projeto de Análise Uber é um projeto de visualização de dados, onde R e as suas bibliotecas são utilizadas para analisar parâmetros ou variáveis como as viagens durante um dia, ou as viagens mensais durante um ano. Estas visualizações para diferentes períodos de tempo anuais são criadas utilizando o ‘Uber Pickups in New York City Dataset’. As bibliotecas e pacotes essenciais de R que precisam de ser importados para este projeto incluem - “ggplot2”, “ggthemes”, “lubridate”, “dplyr”, “tidyr”, “DT”, e “scales”.

3. Sistema de Recomendação de Filmes

Já alguma vez se perguntou como é que o Netflix sugere filmes e séries web dos géneros que lhe atraem instantaneamente? Diferentes plataformas de streaming como Netflix e Amazon Prime utilizam algo conhecido como o Sistema de Recomendação; utiliza um processo de filtragem para sugerir conteúdos com base nas preferências do utilizador, padrões de observação e histórico de navegação. Os dados de navegação do utilizador fornecem a entrada para o Sistema de Recomendações.

Enquanto um Sistema de Recomendação baseado no conteúdo sugere filmes que são semelhantes ao que se viu no passado, a Recomendação de Filtragem Colaborativa fornece sugestões com respeito a outros utilizadores com as mesmas preferências e históricos de visualização. Um Sistema de Recomendação pode ser construído em R usando o “MovieLens Dataset” e os pacotes - “ggplot2”, “recommendenderlab”, “data.table”, e “reshape2”.

4. Segmentação de clientes

A segmentação de clientes é um dos tópicos mais importantes do projeto R. Sempre que as empresas precisam de identificar e visar a base de clientes mais potencial, o método de Segmentação de Clientes vem a calhar. Neste método, a base de clientes é dividida e agrupada de acordo com algumas características semelhantes que são relevantes para o mercado, tais como idade, sexo, interesses, e hábitos de despesa.

É uma forma eficiente para as empresas desenvolverem as suas estratégias de marketing com uma probabilidade mínima de riscos relacionados com o investimento. Os dados recolhidos pelas empresas ajudam-nas a compreender melhor as preferências e requisitos dos clientes individuais que acabam por obter maiores lucros. O projeto de Segmentação de Clientes em R utiliza o algoritmo K significa clustering para agrupar os conjuntos de dados não rotulados e o “Mall Customers Dataset”.

5. Detecção de Fraudes com Cartão de Crédito

A linguagem de programação R encontra outra aplicação na detecção de transacções fraudulentas com cartões de crédito. Neste projeto, são utilizados vários algoritmos de aprendizagem mecânica que podem diferenciar as transacções falsificadas das genuínas. O projeto de detecção de cartões de crédito em R faz uso de múltiplos algoritmos, tais como Regressão Logística, Árvores de Decisão, Classificadores Gradient Boosting, e Redes Neuronais Artificiais.

O conjunto de dados de Transacções com Cartões é utilizado neste projeto de detecção de fraudes com cartões de crédito em R; este conjunto de dados contém transacções fraudulentas, bem como transacções autênticas. O projeto tem as seguintes etapas - importação dos conjuntos de dados contendo as transacções com cartão de crédito, exploração dos dados, manipulação e estruturação dos dados, modelagem dos dados, adaptação do modelo no algoritmo de Regressão Logística, e finalmente, implementação da Árvore de Decisão, Rede Neural Artificial, e modelos de Gradient Boosting.

6. Predição de Preferência de Vinho

A prova de vinhos é uma profissão única em si mesma. Pode ser bastante desafiante prever o que o cliente pode gostar, com base nas suas preferências passadas. Contudo, seria mais fácil para os restaurantes recomendar um vinho aos seus clientes se os seus gostos e preferências fossem previamente identificados; é aqui que o projeto de aprendizagem da máquina R pode ser aplicado. As propriedades físico-químicas do vinho podem ser utilizadas para processos de extracção de dados e identificar as preferências dos clientes. Este projeto particular de aprendizagem da máquina R utiliza o conjunto de dados de qualidade do vinho.

A abordagem adotada no projeto de Previsão das Preferências do Vinho pode ser aplicada a produtos semelhantes para modelar os gostos dos clientes, ajudando assim no marketing alvo. Outra aplicação do R pode ser na previsão da qualidade do vinho, tomando parâmetros físico-químicos como variáveis de entrada para determinar a qualidade do vinho.

**7. Projecto de recomendação musical **

Um sistema de recomendação musical funciona de forma semelhante a um sistema de recomendação de filmes, a única diferença é que, em vez de filmes, recomendará música aos utilizadores. Este é um projecto Python + R. O conjunto de dados utilizado para este projecto é da KKBOX, o principal serviço de transmissão de música na Ásia, com uma biblioteca que contém mais de 30 milhões de faixas musicais.

Neste projecto, será construído um sistema ML usando Python e R que pode prever as hipóteses de um utilizador ouvir uma música em loop após o primeiro evento de audição ter sido desencadeado dentro de uma janela de tempo específica. Aqui, os conjuntos de dados de formação e teste são escolhidos a partir do histórico de audição de diferentes utilizadores num determinado período de tempo.

Assim, por exemplo, se um(s) evento(s) de audição recorrente(s) for desencadeado dentro de um mês após o primeiro evento de audição observável de um utilizador, o sistema marca o alvo como 1 no conjunto de treino, e caso contrário, marca 0. Este projecto é a oportunidade perfeita para aprender a realizar EDA básica para se obterem conhecimentos a partir dos dados.

8. Projecto de identificação do pacote de produtos

O conceito de agrupamento de produtos não é novidade no campo da comercialização. Na abordagem do agrupamento de produtos, os diferentes produtos são agrupados e vendidos como uma única unidade a um preço específico (geralmente preço com desconto). Isto permite aos marqueteiros encorajar os clientes a comprar mais dos seus produtos. Talvez o melhor exemplo de um pacote de produtos seja o McDonald’s Happy Meal.

Neste projecto Data Science, o foco principal será a segmentação subjectiva, uma técnica de clustering que pode ajudar a identificar os melhores pacotes de produtos nos dados de vendas. Aqui, tomaremos um conjunto de dados de transacções de vendas semanais contendo as quantidades adquiridas de diferentes produtos ao longo de algumas semanas.

O conjunto de dados incluirá também valores normalizados. Ao utilizar este conjunto de dados, o objectivo é descobrir quais os produtos que podem ser agrupados para fazer combos excelentes para os clientes. Enquanto a abordagem tradicional utiliza a Análise de Cesto de Mercado para identificar pacotes de produtos, neste projecto, o nosso foco é comparar e analisar a importância relativa do agrupamento de séries temporais na determinação de pacotes de produtos a partir de dados de vendas.
